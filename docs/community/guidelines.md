# AI Kit Community Guidelines

Welcome to the AI Kit community! We're building an inclusive, helpful environment for developers working with AI-powered applications.

## Our Values

1. **Inclusive** - Everyone is welcome regardless of experience level
2. **Helpful** - We support each other's learning and growth
3. **Respectful** - Treat others with kindness and professionalism
4. **Collaborative** - Share knowledge and work together
5. **Innovative** - Experiment, build, and push boundaries

## Code of Conduct

### Expected Behavior

- Be welcoming to newcomers
- Share knowledge generously
- Provide constructive feedback
- Respect differing viewpoints
- Use welcoming and inclusive language
- Accept responsibility for mistakes
- Focus on what's best for the community

### Unacceptable Behavior

- Harassment, trolling, or discriminatory language
- Personal attacks or political arguments
- Publishing others' private information
- Spam, advertising, or excessive self-promotion
- Disruptive behavior in channels or voice chats
- Any conduct that would be unprofessional in a workplace

### Asking for Help

To get the best support:

1. **Search First** - Check documentation and past discussions
2. **Be Specific** - Include code snippets, error messages, versions
3. **Provide Context** - What are you trying to accomplish?
4. **Share Minimal Examples** - Isolate the problem
5. **Format Code** - Use code blocks with syntax highlighting
6. **Be Patient** - Community members volunteer their time
7. **Update Thread** - Mark solved and share solutions

Example help request:
```
I'm trying to implement streaming chat with useAIChat but getting a CORS error.

Environment:
- @ainative/ai-kit: v0.0.1
- Framework: Next.js 14
- Node: 18.17.0

Code:
[paste minimal reproduction]

Error:
[paste error message]

What I've tried:
- Checked API route configuration
- Verified CORS headers
```

### Sharing Projects

When showcasing your work:

- Provide a brief description
- Include screenshots or demo videos
- Share GitHub repo or live demo links
- Explain what AI Kit features you used
- Welcome feedback and questions

### Voice Chat Etiquette

- Mute when not speaking
- Use push-to-talk in busy channels
- Ask before screen sharing
- Keep background noise minimal
- Be respectful of others' time

## Roles and Recognition

### Community Roles

**Member** (Default)
- All community members
- Access to all public channels

**Community Helper** (Yellow)
- Active members who help others
- Earned through consistent helpful participation
- Can moderate help channels

**Contributor** (Green)
- Active code contributors
- Merged pull requests to ai-kit
- Special recognition in community

**Core Team** (Blue)
- Project maintainers and administrators
- AINative Studio team members

### Earning Roles

We recognize valuable community members:

- **Community Helper** - Consistently answer questions in #help
- **Contributor** - Submit quality pull requests
- Roles are granted by Core Team based on participation
- No need to ask - we notice and appreciate you!

## Moderation

### Automated Moderation

Our bot automatically:
- Removes spam (5+ messages/minute)
- Filters inappropriate links
- Limits excessive caps, mentions, emojis
- Welcomes new members

### Human Moderation

Core Team and Community Helpers may:
- Issue warnings for violations
- Temporarily mute disruptive users
- Ban repeat offenders
- Remove inappropriate content

### Reporting Issues

If you see behavior that violates guidelines:
1. Don't engage - let moderators handle it
2. Report using Discord's report feature, or
3. DM a Core Team member directly
4. All reports are confidential

## Contributing to AI Kit

We welcome contributions! See our contribution guidelines:

### Ways to Contribute

1. **Report Bugs** - Use #bug-reports or GitHub Issues
2. **Suggest Features** - Share ideas in #feature-requests
3. **Improve Docs** - Fix typos, clarify explanations
4. **Submit Code** - Fix bugs or implement features
5. **Help Others** - Answer questions in #help
6. **Share Projects** - Inspire others in #showcase

### Contribution Process

1. Check existing issues and discussions
2. Discuss approach in #contributions before major changes
3. Follow the coding standards in the project
4. Write tests for new features
5. Submit PR with clear description
6. Be responsive to code review feedback

For detailed contribution instructions, see [CONTRIBUTING.md](../../CONTRIBUTING.md).
